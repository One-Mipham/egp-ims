## 实施范围

### 第一步：菜单配置重构
- 文件: `frontend/src/config/menuConfig.ts`
- 新增 7 个 `MenuSection`（固定资产/应收/应付/费用报销/进销存/投资/知识库）
- 从会计管理 section 移除已提升的 item
- 调整 section 编号（一~十三）
- 费用报销和知识库设置 `roles` 锁定

### 第二步：路由调整
- 文件: `frontend/src/router/index.ts`
- 提升的模块（投资管理）已有路由，无需改动
- 新增模块（固定资产/应收/应付/进销存/费用报销/知识库）添加占位路由
- 费用报销和知识库路由添加 `allowedRoles` 限制

### 第三步：视图占位
- 为待开发模块创建占位 Vue 页面（显示"模块开发中"）
- 投资管理保持现有页面不变

### 第四步：按优先级开发模块
- ❶ 固定资产管理 → 完整 CRUD + 页面
- ❷ 应收账款 → 完整 CRUD + 页面
- ❸ 应付账款 → 完整 CRUD + 页面
- ❹ 行政综合管理 → 完善现有模块
- ❺ 进销存管理 → 完整 CRUD + 页面

## 设计决策记录

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 费用报销位置 | 独立一级模块 | 有独立审批工作流、预算控制、OCR 识别，非纯记账 |
| 投资管理位置 | 独立一级模块 | 控股公司核心业务，9 子项规模，与已提升模块一致 |
| 知识库 | 占位加锁 | 内容体系待后续规划，先预留位置 |
| 总账是否加费用报销 | 不加 | 总账已 13 项，费用报销独立后不再挤占总账 |

## 约束

- 现有路由路径不修改（避免破坏已有链接和书签）
- 投资管理 9 个子页面路径 `/finance/investments/*` 保持不变
- 会计管理内部编号（十）下的子项编号顺延（9.0~9.7）
- 费用报销和知识库的占位页面显示友好提示，带锁图标
